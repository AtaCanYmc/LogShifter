# Contributing to Logshift

Thank you for your interest in contributing to Logshift! We welcome contributions of all forms, including bug reports, feature requests, documentation improvements, and pull requests.

## How to Contribute

### 1. Reporting Bugs
If you find a bug, please create a GitHub issue and include:
- A clear description of the issue.
- Steps to reproduce the bug.
- The expected behavior.
- Python version, OS version, and dependency versions.

### 2. Suggesting Features
We are open to new transport adapters and core improvements. Please open an issue outlining the feature's purpose, design, and use cases.

### 3. Submitting Pull Requests
- Fork the repository and create your branch from `main`.
- Install dependencies using editable mode: `pip install -e .[dev]`
- Write clear unit tests for any new code.
- Ensure your code follows style guidelines (check with `black` and `ruff`).
- Submit a pull request detailing the changes made.

## Code of Conduct
We expect all contributors to maintain professional, polite, and inclusive communications.
